## Learning Path

### For New Projects

1. **Start with template** (`New_README_Instructions.md`)
2. **Study examples** in `before_after_readme.md`
3. **Apply to your project**
4. **Review with checklist** (`README_Checklist.md`)

### For Existing Projects

1. **Review current README** (`README_Review_Instructions.md`)
2. **Compare with examples**
3. **Identify gaps**
4. **Update sections**
5. **Validate completeness**

## Common Patterns

### Pattern 1: Progressive Disclosure

Start simple, link to details:

```markdown
## Quick Start

```bash
npm install my-tool
npx my-tool init
```

See [Full Installation Guide](docs/installation.md) for advanced options.
```

### Pattern 2: Feature-First

Lead with what makes project unique:

```markdown
## Why This Project?

- **Fast**: 10x faster than alternatives
- **Simple**: One command to get started
- **Flexible**: Extensive plugin system

[Installation](#installation) | [Usage](#usage) | [Docs](docs/)
```
